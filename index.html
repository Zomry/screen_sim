Manual

Introduction

This is a proof-of-concept screen reader simulation developed for Chrome intended to inspire web developers to learn more about assistive technology. It uses the speech synthesis build in Chrome 32+ to give an impression how a screen reader might work. This screen reader was inspired by an interpretation of NVDA and VoiceOver (iOs, OS X). Note though that this is still scratching the surface and that a screen reader is a lot more complex - real websites too. To really start with testing for web accessibility, take some time to learn how a real screenreader works. Some screen readers are: NVDA, VoiceOver, JAWS, Windows Eyes and Supernova.

Features

This simulation supports:

  • Navigation with mouse over on:
    1. Hyperlinks
      • Type: link
      • Text
    2. Ordered and unordered lists
      • Type: ordered or unordered
      • Amount of list items and sub-list items
      • Focus per individual list item
    3. Input-fields, textarea
      • Type: edit
      • Label, if provided
      • Type of input field
      • Value, or if not provided, placeholder
    4. Images (graphic)
      • Type: graphics
      • Says alt-attribute, otherwise file name
      • If longdesc is available, it will give option for that (with enter).
    5. Heading level 1, 2, 3
      • Headings will read their level as part of the accessible name
    6. Table, table heads and table cells
      • Tables will tell how many columns and rows it has.
      • If provided, the table will also read its caption
      • When possible, the table data will give the table head as extra information for the table data
    7. Paragraphs

Navigation

  • ESC: toggle voice on or off
  • k: next hyperlink
  • i: next list item
  • e: next input-field, textarea
  • 1, 2 or 3 for next headers of corresponding level
  • t: next table

Note: all of the above also support previous type of. Press shift-key while typing one of the above commands.
